2. **DAG:** 124-atom prereq graph is acyclic; topo-sort orders atoms for lesson plans.

4. **Stage-gate engine:** 90 unit tests pass; 4 escape valves (timeout / difficulty / cross-track / manual) verified.
5. **Adaptive deck:** rated-5+correct cards retire correctly; difficulty drops on 3 consecutive fails; cross-track lenient mock unlocks at 3-of-4 Qs at S4+.

7. **Pre-flight predictor:** Pearson r=0.93 between first-attempt accuracy and mock score (target ≥0.85).
8. **Source citations:** 100% (2307/2307) of cards carry non-empty `source.kind` + `source.ref`.
9. **Forbidden tokens:** 0 cards contain off-scope keywords (`while`, `do {`, `printf`, `scanf`, `getline`, `recursion`, `recursive`).

## What's deferred to v2.1

### Coverage debt (QA-M10 + QA-M13)

- **6 atoms with 3-5 cards** — pad to ≥6: L-23c, C-30, C-28, C-05, F-22e, F-13. Total ~12 cards to author.
- **33 single-modality atoms** — annotate with `intentionalSingleModality: true` flag OR pad with second card type. Most are intentional speed-only / MCQ-only collections.
- **220 auto-stub CM registry entries** — hand-author full schema (description / whyItHappens / immunizationStrategy / atomDependencies / source) for each.
- **58 CMs with <3 immunization cards** — author missing immunization cards for stub-CMs whose 1-2 referencing cards are insufficient.

### Engineering ticket (QA-M18 / QA-M19 / QA-M20)

- `vite.config.ts` needs `@types/node` in devDependencies. Currently `npm run build` fails at the tsc-noEmit stage due to missing `node:path` and `__dirname` types. Fix: `npm install -D @types/node`.
- E2E (Playwright) not installed. The engine unit tests cover the runtime behavior, but no headless browser fuzz exists for the React shell. Add `@playwright/test` for v2.1.

### UX debt

- The `cpp-t2/docs/v2/UX_DESIGN_SYSTEM.md` documents the design system but the actual screen wiring (Home / Track / Card Screen / Mock Screen / Postmortem) has React components but no top-level routing wired through to `data/v2/cards/` YAML loaders. The `App.tsx` in `src-v2/` exists but isn't the live entrypoint.
- v2.1 should wire `src-v2/App.tsx` to `data/v2/` and replace `src/App.tsx` (the v1 deck). This is a single-day engineering effort once `@types/node` lands.

### What did NOT ship that the spec called for

- Original spec target: 4,600 cards. Shipped: 2,307. Gap: ~50% under target.
  - Root cause: parallel-agent waves authored fewer cards per atom than the cardCountTarget said. Most acute in: L4 (target 700, shipped 381) and L5 mocks (target 250, shipped 86 + 40 mock files = 126 vs 250).
  - Mitigation: the shipped 2,307 cards still exceed the per-atom ≥6 floor for 118/124 atoms (95%). Quality-per-card is high. The "gap" is coverage breadth, not depth.
- Original spec: 30+ specialist agents at peak. Shipped: 6 parallel agents in single 2026-05-06-evening session. The lower-parallelism build still produced a structurally complete deck.
